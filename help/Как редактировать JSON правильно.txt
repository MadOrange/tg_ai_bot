Как редактировать JSON правильно
Всегда используйте двойные кавычки " для ключей и строковых значений.

Экранируйте специальные символы: если внутри строки нужны кавычки " или переносы строк \n, используйте обратный слеш: \", \n.

Проверяйте запятые: после каждого элемента в объекте или массиве (кроме последнего) должна быть запятая.

Используйте валидатор: после правок проверяйте файл на JSONLint.

Способ 1: Заменить переносы строк на специальный символ \n (самый чистый вариант)

json
"description": "* 5 веловылазок по ЗКМ,\n* 18 поездок на работу плюс простые покатушки по Измайловскому парку\n* 300 км по велодорожкам вдоль Балтийского моря.\n* 4 Похода выходного дня (ПВД) Лопатинский рудник, Шемякино → Серпухов, Серпухов → Ступино, Павловский пасад → Дрезна\n* 3 сломанных детали (колесо, покрышка, подседельный штырь) и ни одного прокола!",
Способ 2: Убрать все переносы строк и записать в одну строку (проще для редактирования)

json
"description": "* 5 веловылазок по ЗКМ, * 18 поездок на работу плюс простые покатушки по Измайловскому парку * 300 км по велодорожкам вдоль Балтийского моря. * 4 Похода выходного дня (ПВД) Лопатинский рудник, Шемякино → Серпухов, Серпухов → Ступино, Павловский пасад → Дрезна * 3 сломанных детали (колесо, покрышка, подседельный штырь) и ни одного прокола!",
Способ 3: Использовать массив строк (более структурированно)

json
"description": [
  "5 веловылазок по ЗКМ",
  "18 поездок на работу плюс простые покатушки по Измайловскому парку",
  "300 км по велодорожкам вдля Балтийского моря",
  "4 Похода выходного дня (ПВД): Лопатинский рудник, Шемякино → Серпухов, Серпухов → Ступино, Павловский пасад → Дрезна",
  "3 сломанных детали (колесо, покрышка, подседельный штырь) и ни одного прокола!"
],
